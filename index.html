ES6语法

let/const和var ：命名let当变量需要改变的时候，命名const当变量不需要改变的时候。

var设计是有缺陷的。let可以看成更完美的var。

var没有块级作用域。js中可以用闭包来解决var没有作用域的问题，因为函数就是一个作用域。

ES5之前因为if和for都没有块级作用域的概念，所以在很多时候，我们都必须借助于function的作用域来解决应用外面变量的问题。

ES6中，加入了let，let它有if和for的块级作用域。

promise

四、vue-router路由

前端渲染和后端渲染

后端渲染：用户发请求，后端解析请求，后端类似jsp(java server page)渲染好网页返回给客户端。

后端路由：后端处理URL和网页之间的映射关系。

前端渲染：前后端分离阶段，浏览器中显示的网页中的大部分内容都是由前端写的js代码在浏览器中执行，最终渲染出来的网页。后端专注于数据处理，前端专注于交互和可视化。

前端路由：单页面复应用(SPA)阶段，整个网页只有一个html页面，前端来维护一套路由规则(路由和页面组件匹配)。

综上存在三个阶段：后端渲染阶段->前后端分离阶段->SPA阶段。

五、vuex状态管理

官方解释：状态管理模式，采用集中式存储管理应用的所有组件的状态。

相当于全局单例模式。

一般放到vuex中做共享管理的状态

  • 用户的登录状态、用户名称、token秘钥、头像、地理位置信息等
  • 商品的收藏，购物车的物品等

Vuex中的五个核心概念：

  • State 单一状态树
  • Getters 和计算属性比较像, 有些数据经过变换的到。通过this.$Store.getters.属性 可在多个地方复用。
  • Mutations 传递参数，payload类型（字符串、对象等）
  • Actions 异步操作
  • Module 划分模块，保存数据

vue-CLI2创建项目存在下面两种方式：
runtime-compiler:
template -> ast -> render -> vdom -> UI

runtime-only:
render -> vdom -> UI

结论：runtime-only性能更高，而且代码量更少。

如何选择：如果开发依然用template，就需要用Runtime-Complier；如果选择用.vue开发，那么可以选择Runtime-only。

.vue文件中的template是由vue-template-compiler处理的。

hexo的使用

一：安装Nodejs

node -v //查看node版本
npm -v //查看npm版本
npm install -g cnpm –registry=http://registry.npm.taobao.org //安装淘宝的cnpm 管理器
cnpm -v //查看cnpm版本

二：安装hexo

cnpm install -g hexo-cli //安装hexo框架
hexo -v //查看hexo版本

三：创建博客

mkdir blog-hexo //创建blog目录
cd blog-hexo //进入blog目录
sudo hexo init //初始化博客
hexo s //启动本地博客服务，作预览用 http://localhost:4000/ #本地访问地址
hexo n [我的第一篇文章] //创建文章
#返回blog目录
hexo clean //清理
hexo g //生成

四：部署到github

#Github创建一个新的仓库 YourGithubName.github.io（名称必须是自己的github名）
cnpm install –save hexo-deployer-git //在blog目录下安装git部署插件，如果不通过，前面加sudo。

#设置_config.yml

#回到blog目录

hexo d //部署到远端

输入github的账号和密码，账号名不区分大小写。

五：自定义主题

主题链接：https://github.com/litten/hexo-theme-yilia

按照主题链接下的操作指导进行配置。

六：Hexo Admin插件管理

安装插件

npm install --save hexo-admin

启动服务器
hexo server -d
即可在localhost:4000/admin/中编辑博文了。
